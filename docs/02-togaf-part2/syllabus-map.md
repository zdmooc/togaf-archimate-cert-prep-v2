# TOGAF Enterprise Architecture Part 2 — Syllabus map

## Ce que vous devez savoir faire
- interpréter un scénario
- identifier le contexte exact
- reconnaître la meilleure réponse parmi plusieurs options plausibles
- mobiliser l’ADM dans une situation réelle
- raisonner en termes de gouvernance, parties prenantes, contenu, transitions, capacités

## Ce qu’il faut entraîner
- lire lentement le cas
- repérer le besoin réel
- exclure les réponses techniquement vraies mais contextuellement moins bonnes
- s’appuyer sur le vocabulaire TOGAF exact

## Familles de questions à travailler
- démarrage d’une initiative d’architecture
- adaptation de l’ADM
- architecture vision et engagement des parties prenantes
- gap analysis et planning de migration
- gouvernance et conformité
- gestion des exigences
- content metamodel / artefacts / livrables
- capability-based planning
- architecture repository et continuum

## Exercices recommandés
Pour chaque mini-cas :
1. contexte
2. objectif du sponsor
3. phase(s) ADM dominantes
4. techniques TOGAF mobilisées
5. meilleure réponse
6. pourquoi les autres réponses sont moins bonnes
