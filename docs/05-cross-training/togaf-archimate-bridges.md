# Passerelles TOGAF / ArchiMate

## Idée clé
TOGAF donne la méthode et le cadre.
ArchiMate donne le langage de modélisation.

## TOGAF répond surtout à
- pourquoi architecturer
- comment cadrer
- comment conduire une démarche
- comment gouverner
- quels livrables produire

## ArchiMate répond surtout à
- comment représenter l’architecture
- comment montrer les liens entre métiers, applications et technologies
- comment rendre les vues communicables

## Exemple terrain 1 — migration IBM ODM vers OpenShift
### Avec TOGAF
- cadrage
- parties prenantes
- phases ADM
- transition roadmap
- gouvernance

### Avec ArchiMate
- vue métier des services décisionnels
- vue applicative des composants ODM
- vue technologique de l’hébergement
- vue de migration et work packages

## Exemple terrain 2 — plateforme d’entreprise avec IAM / SSO
### Avec TOGAF
- vision cible
- principes sécurité
- exigences
- analyse d’écart
- roadmap

### Avec ArchiMate
- acteurs
- services d’identité
- applications consommatrices
- flux et relations de service

## Réflexe à installer
À chaque sujet d’étude, se demander :
- quel concept TOGAF est derrière ?
- quelle vue ArchiMate le rend visible ?
