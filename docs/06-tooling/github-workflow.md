# Workflow Git de préparation

## Branching simple
- `main` : dépôt stable
- branches courtes pour les sujets temporaires

## Exemples
- `feat/togaf-p1-concepts`
- `feat/archimate-l1-relations`
- `feat/togaf-p2-open-book`
- `review/mock-exam-01`

## Pull request personnelle
Même seul, ouvrir une PR locale ou brouillon permet de :
- relire
- comparer
- mieux structurer les notes

## Idée de cycle hebdomadaire
- début de semaine : nouvelle branche
- pendant la semaine : notes + tracker
- fin de semaine : synthèse + merge
