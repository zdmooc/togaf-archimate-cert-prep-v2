# Conventions Markdown

## Structure type d’une fiche
- contexte
- définition
- points clés
- erreurs fréquentes
- exemple terrain
- mini-checklist
- questions de révision

## Règles
- une idée principale par section
- phrases courtes
- tableaux quand cela clarifie vraiment
- exemples concrets issus de vos projets quand possible

## Nom des fichiers
- minuscules
- tirets
- noms explicites
